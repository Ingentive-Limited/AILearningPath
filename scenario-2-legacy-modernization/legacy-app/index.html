üîß Legacy Patterns Demonstrated

This application intentionally includes the following legacy patterns that need modernization:

  • Synchronous Operations: All database operations are synchronous
  • Direct Database Access: Controllers directly access the database context
  • No Dependency Injection: Dependencies are created directly in classes
  • Mixed Concerns: Business logic mixed with data access and presentation logic
  • Poor Error Handling: Basic try-catch blocks with minimal error information
  • No Validation: Minimal input validation and business rule enforcement
  • Inefficient Queries: N+1 queries, loading all data into memory
  • No Caching: No response or data caching mechanisms
  • Security Issues: SQL injection vulnerabilities, no authentication
  • Configuration Issues: Hardcoded values, insecure configuration storage
  • No Testing: No unit tests or integration tests
  • Legacy Framework: .NET Framework 4.8, Entity Framework 6

üöÄ Getting Started

To run this legacy application:

  1. Ensure you have .NET Framework 4.8 installed
  2. Restore NuGet packages: nuget restore
  3. Build the solution: msbuild LegacyShop.csproj
  4. Run with IIS Express or deploy to IIS
  5. The database will be created automatically using LocalDB
  6. Sample data will be seeded on first run

üéØ Modernization Goals

Use this application to practice modernizing to:

  • .NET 8: Migrate from .NET Framework 4.8 to .NET 8
  • Entity Framework Core: Migrate from EF6 to EF Core
  • Async/Await: Convert synchronous operations to asynchronous
  • Dependency Injection: Implement proper DI container
  • Clean Architecture: Separate concerns into proper layers
  • Modern C#: Use latest C# language features
  • API Best Practices: Implement proper REST API patterns
  • Security: Add authentication, authorization, and input validation
  • Performance: Implement caching, optimize queries
  • Testing: Add comprehensive unit and integration tests
